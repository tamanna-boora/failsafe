# FAILSAFE — Student Failure Risk Prediction

> **F**ailure **A**lert & **I**ntervention **L**earning **S**ystem for **A**cademic **F**ailure **E**arly-detection

A machine-learning web app that identifies students at risk of academic failure, explains the prediction with SHAP, and recommends personalised interventions — built for the IIT Guwahati Coding Club Even Semester Project 2026.

---

## Problem Statement

Student dropout and academic failure are costly problems for institutions. Traditional grade reviews happen too late. FAILSAFE uses early indicators (study time, attendance, family background, period grades) to flag at-risk students **before** they fail, enabling timely, data-driven interventions.

## Setup & Run

### 1. Clone the repository

```bash
git clone <repo-url>
cd failsafe
```

### 2. Create and activate a virtual environment

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download the dataset

```bash
python download_data.py
```

This fetches the UCI Student Performance dataset (~649 students) and saves it to `data/students.csv`, adding an `at_risk` column (1 = G3 < 10).

### 5. Train the model

```bash
python src/train_model.py
```

Trains an XGBoost classifier, prints accuracy / F1 / confusion matrix, and saves all artifacts to `models/`.

### 6. Launch the dashboard

```bash
streamlit run app/app.py
```

Open `http://localhost:8501` in your browser.

## Key Features

- **Risk Prediction** — XGBoost classifier predicts the probability of a student failing (G3 < 10).
- **Explainable AI** — Per-student SHAP waterfall plots show exactly which features drove the prediction.
- **Personalised Interventions** — Rule-based engine generates 3–5 targeted action recommendations based on the student's top SHAP features.
- **Interactive Dashboard** — Filter by risk level (High / Medium / Low), view summary stats, and drill down into individual student profiles.
- **CSV Upload** — Educators can upload their own student data for batch prediction.

---

## Dataset

[UCI Student Performance Data Set](https://archive.ics.uci.edu/dataset/320/student+performance) — collected from two Portuguese secondary schools. Features include demographics, family background, study habits, extracurricular activities, alcohol consumption, and first/second period grades.

**Target:** `at_risk = 1` if final grade G3 < 10 (out of 20), else 0.

## Future Scope

The current prototype is intentionally simple for a 4-day deadline. A production-grade version would include:

- **Backend:** FastAPI REST API replacing the in-process Streamlit inference
- **Database:** PostgreSQL for persistent student records, predictions, and intervention logs
- **Authentication:** JWT-based role authentication (admin / teacher / counsellor)
- **Frontend:** React + TypeScript SPA with interactive charts (Recharts / D3)
- **Model Registry:** MLflow for experiment tracking and model versioning
- **Retraining Pipeline:** Scheduled retraining as new grade data arrives
- **Alerting:** Email / SMS notifications for counsellors when students cross risk thresholds
- **Multi-institution Support:** Multi-tenant architecture with per-school models
